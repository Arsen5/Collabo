# Collabo — канбан-система для управления задачами

Современная канбан-доска с real-time обновлениями, авторизацией, общими досками и автоматизацией.

## Демо

http://186.246.30.98:5000/html/auth.html

(Если ссылка недоступна, сервер мог быть остановлен. Для локального запуска следуйте инструкции ниже.)

## Функциональность

- Канбан-доска (To-Do, In Progress, Done)
- Real-time синхронизация через SignalR
- Авторизация (JWT)
- Доски (создание, удаление, переключение)
- Приглашение пользователей в доски
- Задачи (создание, редактирование, удаление, Drag & Drop)
- Автоматизация: просрочка дедлайна, тег "urgent" → high priority
- Фильтрация задач (по приоритету, статусу, поиск)
- Экспорт досок в JSON и CSV

## Архитектура
-----------------------------------------------------------------------------------
Уровень 1: Клиент (Фронтенд)
HTML страницы: auth.html (вход/регистрация), board.html (канбан-доска), create-board.html (создание доски)

JavaScript модули:

api-integration.js - все запросы к API и подключение к SignalR

sidebar-navigation.js - загрузка и отображение списка досок в меню

script.js - UI логика (модальные окна, drag & drop, правый сайдбар)

CSS стили: оформление доски, колонок, карточек задач, модальных окон
------------------------------------------------------------------------------------
Уровень 2: Веб-сервер (ASP.NET Core)
REST API:

Авторизация: /api/auth/register, /api/auth/login

Доски: /api/boards (GET, POST, DELETE), /api/user/boards (свои + приглашённые)

Задачи: /api/tasks (GET, POST, PUT, DELETE), /api/boards/{id}/tasks (задачи конкретной доски)

Приглашения: /api/boards/{id}/invite

SignalR Hub (/tasksHub): real-time рассылка событий о создании, обновлении, удалении задач

JWT аутентификация: проверка токена при каждом запросе, редирект на auth.html при 401 ошибке
------------------------------------------------------------------------------------
Уровень 3: Бизнес-логика
Автоматизация:

При создании/обновлении задачи проверяется тег "urgent" → приоритет повышается до "high"

При просрочке дедлайна карточка задачи подсвечивается красным

При высоком приоритете показывается всплывающее уведомление

Очередь событий (RabbitMQ / MassTransit):

При удалении задачи событие отправляется в очередь

Consumer обрабатывает событие (логирование, уведомления и т.д.)

Фильтрация: поиск по названию, фильтры по приоритету и статусу

Экспорт: выгрузка задач текущей доски в JSON или CSV
------------------------------------------------------------------------------------
Уровень 4: База данных (PostgreSQL)
AspNetUsers — пользователи (Identity)

Boards — доски (Id, Name, OwnerId, CreatedAt)

BoardMembers — приглашения (BoardId, UserId, Role)

Tasks — задачи (Id, Title, Description, Status, Priority, Tags, DueDate, Assignee, BoardId, CreatedAt)

__EFMigrationsHistory — история миграций

Потоки данных
Вход в систему:

Пользователь отправляет email/пароль → сервер проверяет → возвращает JWT токен → токен сохраняется в localStorage → редирект на board.html

Загрузка досок:

Frontend отправляет GET /api/user/boards с токеном → сервер возвращает свои доски + доски с приглашениями → отображение в левом меню

Создание задачи:

Пользователь заполняет форму → отправляется POST /api/tasks → задача сохраняется в БД → SignalR рассылает событие всем клиентам → клиенты перезагружают задачи

Real-time синхронизация:

SignalR поддерживает WebSocket соединение → при любом изменении задач сервер отправляет событие → все клиенты с той же доской получают уведомление и обновляют UI

Приглашение в доску:

Владелец вводит email → отправляется POST /api/boards/{id}/invite → сервер находит пользователя → добавляет запись в BoardMembers → приглашённый пользователь видит доску после перезагрузки

Автоматизация:

При создании/обновлении задачи проверяются правила → если тег "urgent", приоритет меняется на "high" → если дедлайн просрочен, карточка подсвечивается красным → уведомление отображается на экране

## Требования

- .NET 10 SDK
- PostgreSQL 16 или 17
- Docker (опционально, для RabbitMQ)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/Arsen5/Collabo.git
cd Collabo
```
2. Настройка базы данных
Установите PostgreSQL и создайте базу данных:
```sql
CREATE DATABASE taskflow;
```
3. Установка инструментов Entity Framework
```bash
dotnet tool install --global dotnet-ef
```
4. Создание и применение миграций
```bash
dotnet ef migrations add InitialCreate
dotnet ef database update
```
5. Запуск RabbitMQ (через Docker)
```bash
docker-compose up -d
```
6. Запуск проекта
```bash
dotnet run
```
7. Открытие в браузере
```text
http://localhost:5000/html/auth.html
```
Структура проекта
```text
Collabo/
├── Models/              # Модели данных (TaskItem, Board, User, BoardMember)
├── Data/                # DbContext и настройки Entity Framework
├── Hubs/                # SignalR Hub для real-time
├── hak/                 # Фронтенд (HTML, CSS, JavaScript)
│   ├── css/             # Стили
│   ├── html/            # Страницы (auth, board, create-board)
│   └── js/              # JavaScript (API, навигация, скрипты)
├── Program.cs           # Точка входа и конфигурация API
└── docker-compose.yml   # Настройка RabbitMQ
```

Автоматизация
При добавлении тега "urgent" приоритет задачи автоматически повышается до "high"

При просрочке дедлайна задача подсвечивается красным

При создании задачи с высоким приоритетом показывается уведомление

Возможные проблемы и решения
Ошибка подключения к базе данных
Проверьте, запущен ли PostgreSQL

Проверьте правильность пароля в строке подключения

Выполните: 
```bash
dotnet ef database update
```

Порт 5000 уже используется
Измените порт в Program.cs:
```csharp
builder.WebHost.UseUrls("http://0.0.0.0:5001");
```
Ошибка 401 при запросах
Убедитесь, что вы залогинены и передаёте токен в заголовке Authorization: Bearer <token>.
