# Collabo
СИСТЕМА ДЛЯ УПРАВЛЕНИЯ ЗАДАЧАМИ

Описание задачи: разработать канбан-систему, которая:
 — работает в реальном времени
 — обрабатывает поток задач
 — поддерживает автоматизацию событий
Система должна имитировать продукт уровня Jira/Trello с event-driven архитектурой.

Цель: создать масштабируемую систему, которая:
 — отображает задачи на канбан-доске
 — синхронизирует изменения между пользователями
 — автоматически реагирует на события

Целевая аудитория:
 — Пользователи: сотрудники и команды
 — Администраторы: менеджеры и организаторы процессов

Основной функционал:
Для пользователей:
 — просмотр и управление задачами (создание, редактирование, удаление, перемещение)
 — уведомления о событиях в реальном времени
Для администраторов:
 — настройка колонок и правил автоматизации
 — контроль входящего потока задач
 — управление уведомлениями

Функциональные требования:
 — Канбан-доска: To Do, In Progress, Done, кастомные колонки
 — Карточка задачи: ID, название, описание, статус, приоритет, теги, дата создания, дедлайн
 — События: создание, обновление, перемещение, удаление задачи
 — Автоматизация: уведомления, перемещение задач по правилам, флаги по тегам
 — Очередь задач: обработка через API/очередь (RabbitMQ/Kafka), дедубликация, валидация, обогащение
 — Уведомления: системные и пользовательские в real-time

Нефункциональные требования:
 — Надёжность: сообщения не теряются, ошибки обрабатываются
 — Консистентность: согласованное отображение изменений, контроль конфликтов

Ожидаемый результат:
 — Рабочее приложение (backend ± frontend)
 — Исходный код
 — README с архитектурой, технологиями, инструкцией по запуску
 — Развертывание на сервере
